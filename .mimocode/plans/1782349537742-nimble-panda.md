# System Settings API Implementation Plan

## Goal
Add runtime-configurable settings (TMDB key, proxy, adult content toggle) via REST API, so users can configure from frontend without editing config files.

## Architecture

### New Files (in `common` module)

1. **`common/src/main/java/com/fryfrog/hub/common/model/SystemSetting.java`**
   - JPA entity extending `BaseEntity`
   - Fields: `key` (String, unique), `value` (String), `description` (String)
   - Table name: `system_setting`

2. **`common/src/main/java/com/fryfrog/hub/common/repository/SystemSettingRepository.java`**
   - `JpaRepository<SystemSetting, Long>`
   - `Optional<SystemSetting> findByKey(String key)`

3. **`common/src/main/java/com/fryfrog/hub/common/service/SystemSettingService.java`**
   - In-memory cache (`ConcurrentHashMap`) for fast reads
   - `getValue(key, defaultValue)` - returns value or default
   - `getBoolean(key, defaultValue)` - for boolean settings
   - `getDouble(key, defaultValue)` - for numeric settings
   - `getInteger(key, defaultValue)` - for integer settings
   - `setValue(key, value, description)` - save to DB + update cache
   - `getAll()` - return all settings
   - `@PostConstruct` init: load all from DB into cache

### New Files (in `app` module)

4. **`app/src/main/java/com/fryfrog/hub/controller/SettingController.java`**
   - `GET /api/v1/settings` - list all settings
   - `GET /api/v1/settings/{key}` - get one setting
   - `PUT /api/v1/settings/{key}` - update setting value
   - `GET /api/v1/settings/tmdb/status` - check TMDB config status

### Files to Modify

5. **`video/src/main/java/com/fryfrog/hub/video/service/TmdbService.java`**
   - Inject `SystemSettingService`
   - Replace `@Value` fields with calls to `SystemSettingService.getValue()`
   - Make `include_adult` configurable via setting `tmdb.include-adult`

6. **`video/src/main/java/com/fryfrog/hub/video/service/VideoService.java`**
   - Inject `SystemSettingService`
   - Replace `@Value` for `autoScrape`, `minScore` with settings service

7. **`common/src/main/java/com/fryfrog/hub/common/config/RestTemplateConfig.java`**
   - Inject `SystemSettingService`
   - Replace `@Value` for proxy config with settings service
   - Note: Proxy changes require app restart (RestTemplate is a singleton bean)

8. **`comic/src/main/java/com/fryfrog/hub/comic/service/MangaScrapeService.java`**
   - Replace `@Value` for `autoScrape`, `minScore` with settings service

9. **`comic/src/main/java/com/fryfrog/hub/comic/service/AnilistService.java`**
   - Replace `@Value` for `language` with settings service

## Settings Registry

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `tmdb.api-key` | string | (empty) | TMDB API Key |
| `tmdb.language` | string | `zh-CN` | TMDB language |
| `tmdb.image-size` | string | `original` | TMDB image size |
| `tmdb.auto-scrape` | boolean | `false` | Auto scrape on scan |
| `tmdb.min-score` | double | `0.0` | Min score threshold |
| `tmdb.include-adult` | boolean | `true` | Include adult content |
| `proxy.host` | string | (empty) | Proxy host |
| `proxy.port` | integer | `0` | Proxy port |
| `anilist.auto-scrape` | boolean | `false` | Auto scrape comics |
| `anilist.min-score` | double | `0.0` | Anilist min score |

## API Endpoints

```
GET    /api/v1/settings              - List all settings
GET    /api/v1/settings/{key}        - Get setting by key
PUT    /api/v1/settings/{key}        - Update setting { "value": "xxx" }
GET    /api/v1/settings/tmdb/status  - TMDB config status check
```

## Implementation Order

1. Create `SystemSetting` entity + repository + service in `common`
2. Create `SettingController` in `app`
3. Update `TmdbService` (API key, language, include_adult)
4. Update `VideoService` (auto-scrape, min-score)
5. Update `RestTemplateConfig` (proxy) - note restart required
6. Update `MangaScrapeService` and `AnilistService`
7. Test with `mvn clean install`

## Limitations

- **Proxy config**: Changing proxy host/port at runtime won't take effect until app restart, because `RestTemplate` is created as a singleton bean at startup. We can log a warning when these are changed.
- **Config file values**: The `@Value` defaults in `application.yml` serve as initial values. Once a setting is stored in DB, the DB value takes precedence.
