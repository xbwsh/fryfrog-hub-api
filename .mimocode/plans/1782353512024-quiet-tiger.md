# 漫画文件夹结构规划 + 角色图片本地化

## 目标

1. 规划漫画系列文件夹结构，统一管理封面、角色图片等元数据
2. 刮削角色时下载图片到本地，不依赖远程 URL

## 当前状态

- 封面已改为存到漫画同目录（上一轮改动）
- 角色图片只存远程 URL（`imageUrl` 字段），不下载
- `moveComicToSeriesFolder` 把漫画文件移到系列文件夹，但没有组织附属文件

## 目标文件夹结构

```
media-library/comic/
└── SPY×FAMILY/
    ├── SPY×FAMILY Vol.1.cbz
    ├── SPY×FAMILY Vol.1_cover.jpg      # 单行本封面
    ├── SPY×FAMILY Vol.2.cbz
    ├── SPY×FAMILY Vol.2_cover.jpg
    ├── series_cover.jpg                 # 系列封面（从数据源下载）
    └── characters/                      # 角色图片目录
        ├── loid.jpg
        ├── anya.jpg
        └── yor.jpg
```

## 修改文件清单

### 1. `ComicCharacter.java` — 新增 `imagePath` 字段

新增本地图片路径字段，刮削时下载图片后记录路径。

```java
@Schema(description = "角色图片本地路径")
private String imagePath;
```

### 2. `MangaScrapeService.java` — 下载角色图片 + 系列封面

**`saveBangumiCharacters()`** — 每个角色下载图片到 `{seriesDir}/characters/{角色名}.jpg`

**`saveAnilistCharacters()`** — 同上

**`downloadCover()`** — 增加系列封面下载：除了单行本封面外，也下载一张 `series_cover.jpg`

新增方法：
```java
private String downloadCharacterImage(Long comicId, String imageUrl, String characterName)
```
- 存储路径：`{comicDir}/characters/{sanitizedName}.jpg`
- 返回本地路径

### 3. `MangaScrapeService.moveComicToSeriesFolder()` — 同步移动角色图片

当前只移动封面和缩略图。需要增加：如果角色图片存在，也移动到新的系列文件夹。

### 4. `ComicController.java` — 角色图片接口

新增获取角色本地图片的接口：
```
GET /api/v1/comic/character/{characterId}/image
```
返回本地图片文件，如果本地没有则 302 到远程 `imageUrl`。
