# Hanime 刮削器集成到 video 模块

## 概述

将 Hanime 元数据刮削功能集成到现有的 video 模块中，作为 TMDB 之外的第二个元数据来源。遵循项目 YAGNI 风格，不引入额外的接口抽象层。

## 新增文件

| 文件 | 说明 |
|------|------|
| `video/src/main/java/com/fryfrog/hub/video/service/HanimeScraperService.java` | 核心刮削服务，Jsoup 解析 HTML |
| `video/src/main/java/com/fryfrog/hub/video/service/CfBypassClient.java` | Cloudflare 绕过 HTTP 客户端 |
| `video/src/main/java/com/fryfrog/hub/video/util/ChineseConverter.java` | 繁体→简体转换工具 |
| `video/src/main/java/com/fryfrog/hub/video/dto/HanimeMetadata.java` | 刮削结果 DTO |

## 修改文件

| 文件 | 变更 |
|------|------|
| `video/pom.xml` | 添加 jsoup、opencc4j 依赖 |
| `video/.../model/Video.java` | 添加 hanimeId、tags、viewCount、studio、subtitle 字段 |
| `video/.../service/VideoService.java` | 新增 hanime 刮削入口方法 |
| `video/.../controller/VideoController.java` | 新增 hanime 相关 REST 端点 |
| `app/src/main/resources/application.yml` | 添加 hanime 配置段 |

## 详细设计

### 1. video/pom.xml — 新增依赖

```xml
<dependency>
    <groupId>org.jsoup</groupId>
    <artifactId>jsoup</artifactId>
    <version>1.17.2</version>
</dependency>
<dependency>
    <groupId>org.cnough</groupId>
    <artifactId>opencc4j</artifactId>
    <version>1.2.1</version>
</dependency>
```

### 2. Video.java — 新增字段

在现有 `metadataSource` 字段附近添加：

```java
// Hanime ID
private String hanimeId;

// 标签（逗号分隔）
@Column(columnDefinition = "TEXT")
private String tags;

// 观看次数
private Integer viewCount;

// 制作商
private String studio;

// 副标题
private String subtitle;
```

### 3. HanimeMetadata.java — DTO

```java
@Data @Builder @NoArgsConstructor @AllArgsConstructor
public class HanimeMetadata {
    private String videoId;
    private String title;
    private String subtitle;
    private String description;
    private String coverUrl;
    private String studio;
    private String videoType;
    private String uploadDate;
    private Integer viewCount;
    private List<String> tags;
}
```

### 4. CfBypassClient.java — CF 绕过客户端

- 通过 `RestTemplate` 调用 cf-bypass Docker 服务
- 配置：`hub.hanime.cf-bypass-url`（cf-bypass 服务地址）
- 支持代理：`hub.hanime.use-proxy` + `hub.proxy.host/port`
- 检测 CF 挑战页面特征码自动重试
- 请求头：`x-hostname`、`x-proxy`、`x-bypass-cache`

### 5. HanimeScraperService.java — 核心刮削

- 使用 Jsoup 解析 hanime1.me 页面 HTML
- Caffeine 缓存（60分钟TTL，1000条上限）
- 请求间隔控制（默认1500ms）
- 支持单个刮削和批量刮削
- 标题繁简转换（opencc4j）
- 读取配置：`hub.hanime.base-url`、`hub.hanime.scraper.*`

核心方法：
```java
public HanimeMetadata scrape(String videoId)           // 单个刮削
public Map<String, HanimeMetadata> scrapeBatch(List<String> ids)  // 批量
```

### 6. ChineseConverter.java — 繁简转换

- 使用 opencc4j 库（`com.github.huan3d.opencc4j`）
- 提供 `toSimplified(String traditional)` 方法
- Hanime 页面为繁体中文，刮削后转简体存入数据库

### 8. VideoController.java — 新增端点

```
POST /api/v1/video/{id}/hanime/bind?hanimeId=xxx
  → 绑定 Hanime 元数据到指定视频

POST /api/v1/video/hanime/scrape?hanimeId=xxx&downloadCover=true&generateNfo=true
  → 仅刮削，不绑定到已有视频（返回元数据）
```

### 9. application.yml — 新增配置

```yaml
hub:
  hanime:
    base-url: ${HANIME_BASE_URL:https://hanime1.me}
    cf-bypass-url: ${HANIME_CF_BYPASS_URL:http://localhost:8000}
    use-proxy: ${HANIME_USE_PROXY:false}
    scraper:
      request-interval: ${HANIME_REQUEST_INTERVAL:1500}
      max-retries: ${HANIME_MAX_RETRIES:3}
      timeout: ${HANIME_TIMEOUT:30}
```

## 实现顺序

1. 添加 Maven 依赖（jsoup、opencc4j）
2. 扩展 Video 实体字段
3. 实现 ChineseConverter 工具类
4. 实现 HanimeMetadata DTO
5. 实现 CfBypassClient
6. 实现 HanimeScraperService
7. 在 VideoService 中添加 hanime 绑定方法
8. 在 VideoController 中添加 REST 端点
9. 添加 application.yml 配置

## 验证方式

1. `mvn compile -pl video` — 编译通过
2. `mvn compile -pl app` — 全模块编译通过
3. 启动应用，访问 Swagger UI (`/swagger-ui.html`) 确认新端点
4. 如已部署 cf-bypass Docker 服务，可调用刮削 API 测试
