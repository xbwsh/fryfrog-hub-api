# 封装 FFmpeg：通过 JavaCV 内嵌原生二进制

## 背景

当前项目通过 `ProcessBuilder` 调用系统 `ffprobe`/`ffmpeg`，要求用户手动安装并配置 PATH。目标是将 FFmpeg 原生二进制打包进应用，开箱即用。

## FFmpeg 使用现状

所有调用集中在 `MediaInfoService.java`（3 处 ProcessBuilder）：

| 方法 | 工具 | 用途 |
|------|------|------|
| `analyzeFile()` | ffprobe | 分析视频技术信息（编码、分辨率、帧率等） |
| `extractSubtitleAsVtt()` | ffmpeg | 提取内嵌字幕转 WebVTT |
| `convertExternalSubtitleToVtt()` | ffmpeg | 外挂字幕转 WebVTT |

配置项：`hub.video.ffprobe-path`（默认 `ffprobe`）和 `hub.video.ffmpeg-path`（默认 `ffmpeg`）。

## 方案

使用 `org.bytedeco:ffmpeg-platform` 依赖内嵌 FFmpeg 原生二进制，保留现有 ProcessBuilder 调用方式不变（改动最小）。

### 步骤 1：添加 Maven 依赖

**文件**: `video/pom.xml`

```xml
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>ffmpeg-platform</artifactId>
    <version>6.1-1.5.10</version>
</dependency>
```

不加 `<classifier>`，会拉取所有平台（windows-x86_64、linux-x86_64、macosx-x86_64 等），JAR 体积约 200-300MB。如需减小可加 classifier 只打包目标平台。

### 步骤 2：创建 FFmpeg 二进制提取工具类

**新建文件**: `video/src/main/java/com/fryfrog/hub/video/util/FfmpegBinaryExtractor.java`

职责：
- 启动时检测当前 OS（windows/linux/macos）
- 从 classpath 中 bytedeco 的 ffmpeg-platform JAR 里提取 `ffmpeg` 和 `ffprobe` 到临时目录（如 `{java.io.tmpdir}/fryfrog-ffmpeg/`）
- 提取前检查是否已存在（避免重复解压）
- 返回提取后的 ffmpeg 和 ffprobe 的完整路径

实现要点：
- 用 `Loader.load(org.bytedeco.ffmpeg.global.Ffmpeg.class)` 或直接从 JAR 中读取 native resource
- Windows 下二进制名为 `ffmpeg.exe` / `ffprobe.exe`，其他平台无扩展名
- 提取目录用 `{java.io.tmpdir}/fryfrog-ffmpeg/{os}-{arch}/`

### 步骤 3：修改 MediaInfoService 启动时自动配置路径

**文件**: `video/src/main/java/com/fryfrog/hub/video/service/MediaInfoService.java`

改动：
- 在 `@PostConstruct` 或构造函数中，先尝试系统 PATH 中的 `ffprobe`/`ffmpeg`
- 如果不存在（启动时快速检测），调用 `FfmpegBinaryExtractor` 提取内嵌二进制
- 用提取的路径覆盖 `ffprobePath` / `ffmpegPath` 字段
- 启动日志输出使用的是系统版本还是内嵌版本

### 步骤 4：application.yml 添加可选配置覆盖

**文件**: `app/src/main/resources/application.yml`

保持现有配置项不变（`hub.video.ffprobe-path` / `hub.video.ffmpeg-path`），用户仍可手动指定路径覆盖自动检测。

## 不需要改动的地方

- 所有 ProcessBuilder 调用逻辑不变（命令行参数、超时处理、错误处理）
- VideoService 中的 `mediaInfoService.updateVideoMediaInfo()` 调用不变
- VideoController 中的 media-info、subtitles、subtitle/vtt 端点不变
- 其他模块（music/comic/ebook）不涉及 FFmpeg，无需改动

## 涉及的文件

| 文件 | 操作 |
|------|------|
| `video/pom.xml` | 添加 ffmpeg-platform 依赖 |
| `video/.../util/FfmpegBinaryExtractor.java` | **新建** — 二进制提取工具 |
| `video/.../service/MediaInfoService.java` | 修改 — 启动时自动检测/提取 |

## 验证方式

1. `mvn compile` 编译通过
2. 启动应用，观察日志是否输出"使用内嵌 FFmpeg"（而非 ffprobe not found 警告）
3. 扫描视频目录，验证 ffprobe 分析成功（不再报 `Cannot run program "ffprobe"` 错误）
4. 调用 `GET /api/v1/video/{id}/media-info` 返回完整的编码/分辨率信息
