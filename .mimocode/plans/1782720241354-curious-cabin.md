# 多路径媒体库 + 类型分类精准刮削

## 目标

支持多路径资源库，每个库有类型（movie/tv/mixed），扫描时自动关联库类型，TMDB 刮削时根据类型精准搜索，避免同标题不同版本的误刮削（如堀与宫村 TV vs 电影）。

## 方案概述

### 1. 新建 MediaLibrary 实体

```java
@Entity
@Table(name = "media_libraries")
public class MediaLibrary extends BaseEntity {
    String name;           // 库名称，如 "电影库"、"日剧"
    String path;           // 文件系统路径
    String mediaType;      // "movie" / "tv" / "mixed"
    String description;    // 描述
    Boolean enabled;       // 是否启用扫描
    Integer sortOrder;     // 排序
}
```

**关键文件**: `common/src/main/java/com/fryfrog/hub/common/model/MediaLibrary.java` (新建)

### 2. 新建 MediaLibrary CRUD

- `MediaLibraryRepository` — 基础 CRUD + findByMediaType
- `MediaLibraryService` — CRUD + getPathList(enabled=true)
- `MediaLibraryController` — `GET/POST/PUT/DELETE /api/v1/media-libraries`

**关键文件**:
- `common/src/main/java/com/fryfrog/hub/common/repository/MediaLibraryRepository.java` (新建)
- `common/src/main/java/com/fryfrog/hub/common/service/MediaLibraryService.java` (新建)
- `common/src/main/java/com/fryfrog/hub/common/controller/MediaLibraryController.java` (新建)

### 3. 修改 VideoController — 去掉 @Value root-paths

当前 `VideoController.java:66-74` 从 `@Value("${hub.video.root-paths}")` 读取路径列表。改为从 `MediaLibraryService` 读取所有 enabled 的视频库路径。

```java
// Before
@Value("${hub.video.root-paths:./media-library/video}")
private String rootPathsConfig;

// After
private final MediaLibraryService mediaLibraryService;

private List<String> getRootPaths() {
    return mediaLibraryService.getEnabledPaths("video");
}
```

同时修改 `rescan()` 方法，传入每个库的 mediaType。

**关键文件**: `video/src/main/java/com/fryfrog/hub/video/controller/VideoController.java`

### 4. 修改 VideoService — 根据库类型精准搜索

#### 4a. searchFromTmdb 增加 mediaType 参数

```java
// Before: 始终合并 movie + TV
public List<TmdbSearchResult.TmdbSearchItem> searchFromTmdb(String query)

// After: 根据 mediaType 只搜对应类型
public List<TmdbSearchResult.TmdbSearchItem> searchFromTmdb(String query, String mediaType)
```

- `mediaType = "movie"` → 只调 `tmdbService.searchMovies()`
- `mediaType = "tv"` → 只调 `tmdbService.searchTv()`
- `mediaType = null` 或 `"mixed"` → 保持当前合并行为

#### 4b. tryScrapeVideo 传入 mediaType

```java
// Before
private void tryScrapeVideo(Video video)

// After
private void tryScrapeVideo(Video video, String libraryMediaType)
```

调用 `searchFromTmdb(query, libraryMediaType)` 而非 `searchFromTmdb(query)`。

#### 4c. autoScrapeAll 支持按库类型批量刮削

新增 `autoScrapeByLibrary(MediaLibrary library)` 方法，或在现有 `autoScrapeAll()` 中遍历所有库并传入类型。

#### 4d. scanDirectory 记录库类型到视频

扫描时将库的 mediaType 写入 Video 实体（如果 video.mediaType 为空），作为 fallback。

**关键文件**: `video/src/main/java/com/fryfrog/hub/video/service/VideoService.java`

### 5. Video 实体 — mediaType 与库类型的关系

- 扫描时：如果 video.mediaType 为 null，从所属库的 mediaType 填充
- 刮削时：优先用库类型，库类型为 mixed 时用当前合并逻辑
- 绑定后：video.mediaType 被 TMDB 结果覆盖（已是当前行为）

### 6. 迁移策略

- 新增 `media_libraries` 表
- 数据库迁移脚本：将当前 `hub.video.root-paths` 配置中的路径自动导入为默认库（mediaType=mixed）
- 保留 `hub.video.root-paths` 配置作为 fallback（MediaLibrary 表为空时使用）

### 7. 前端需要的接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/media-libraries` | GET | 获取所有库 |
| `/api/v1/media-libraries` | POST | 创建库 |
| `/api/v1/media-libraries/{id}` | PUT | 更新库 |
| `/api/v1/media-libraries/{id}` | DELETE | 删除库 |
| `/api/v1/media-libraries/video/paths` | GET | 获取视频模块所有启用路径（供扫描使用） |

## 实现顺序

1. MediaLibrary 实体 + Repository + Service + Controller（common 模块）
2. 数据库迁移脚本（从配置导入默认库）
3. 修改 VideoController — 从 MediaLibrary 读路径
4. 修改 VideoService.searchFromTmdb — 增加 mediaType 参数
5. 修改 tryScrapeVideo / autoScrapeAll — 传入库类型
6. 修改 scanDirectory — 扫描时填充 video.mediaType
7. 测试验证

## 验证方式

1. 创建两个库：电影库 (movie) + 电视剧库 (tv)
2. 分别扫描放入堀与宫村电影版和 TV 版
3. 触发自动刮削，验证电影库只搜 movie、电视剧库只搜 tv
4. 手动搜索 `GET /api/v1/video/tmdb/search?q=堀与宫村` 验证类型过滤
5. 混合库 (mixed) 保持当前合并行为
