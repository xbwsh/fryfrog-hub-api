# 修复音频转码播放问题

## 问题诊断

"彻底播放不了"的根因链：

1. FFmpeg 转码失败（具体原因未知，因为 stderr 被 `DISCARD` 丢弃）
2. `transcodeAudio()` 抛出 `RuntimeException`
3. 异常传播到 `GlobalExceptionHandler`
4. 异常处理器返回 **JSON** 格式的错误响应（`ApiResponse.error("Internal server error")`）
5. 浏览器期望收到 `video/mp4` 视频流，却收到 JSON → `<video>` 报错 "no supported sources"

## 修复方案

### 1. 捕获 FFmpeg 错误日志（定位真实原因）

**文件**: `MediaInfoService.java` 的 `transcodeAudio()` 方法

- 将 `pb.redirectError(ProcessBuilder.Redirect.DISCARD)` 改为读取 stderr 并打日志
- 转码失败时日志输出 FFmpeg 的完整错误信息

### 2. 给临时文件名加 UUID（防并发冲突）

**文件**: `MediaInfoService.java`

- 输出路径改为 `/tmp/fryfrog-transcode/{UUID}_{baseName}.mp4`
- 避免多个视频同时转码时文件互相覆盖

### 3. 在 streamVideo 中 try-catch 转码异常（防 500 JSON 响应）

**文件**: `VideoController.java` 的 `streamVideo()` 方法

- try-catch 包裹 `transcodeAudio()` 调用
- 转码失败时 fallback 到直接播放原始文件（音频可能不兼容但至少能播放）
- 记录警告日志

### 4. 替换 deleteOnExit 为定时清理（长期运行服务的磁盘安全）

**文件**: `MediaInfoService.java`

- 新增 `@Scheduled` 方法，每小时清理 `/tmp/fryfrog-transcode/` 中超过 1 小时的文件
- 移除 `videoFile.deleteOnExit()`

### 5. 缓存 ffprobe 结果（避免每次播放都调 ffprobe）

**文件**: `VideoController.java` / `MediaInfoService.java`

- 用 Caffeine Cache 缓存 ffprobe 结果，key=视频文件路径，TTL=5 分钟
- 同一视频多次请求不重复调 ffprobe

## 涉及文件

1. `video/src/main/java/com/fryfrog/hub/video/service/MediaInfoService.java`
   - `transcodeAudio()`: 加 stderr 日志 + UUID 文件名
   - 新增 `cleanupTempTranscodes()` 定时清理方法

2. `video/src/main/java/com/fryfrog/hub/video/controller/VideoController.java`
   - `streamVideo()`: try-catch 转码异常 + fallback

## 验证方式

1. 启动服务，用 EAC3 音频的视频（如铁拳教育 S01E01）测试 `GET /api/v1/video/6/stream`
2. 检查日志：如果有 FFmpeg 错误，stderr 内容应该会打印出来
3. 转码成功时：进度条正常、可 seek、音画同步
4. 转码失败时：fallback 播放原始文件（有画面无声音或有报错但不白屏）
