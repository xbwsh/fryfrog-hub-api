# 视频音频转码方案

## 需求
Electron 客户端播放 MKV 视频时，EAC3/DTS/TrueHD 等音频编码浏览器不支持，需要后端用 FFmpeg 实时转码为 AAC。

## 现状
- `VideoController.streamVideo` 直接返回 `FileSystemResource`，无转码
- 项目中无 FFmpeg、ProcessBuilder 相关代码
- 无 Range 请求支持

## 方案

### 1. 配置 FFmpeg 路径

`application.yml` 新增：
```yaml
hub:
  ffmpeg:
    path: ${FFMPEG_PATH:ffmpeg}
  video:
    transcode-audio: ${VIDEO_TRANSCODE_AUDIO:true}
```

### 2. 新增 FfmpegService

`video/src/main/java/com/fryfrog/hub/video/service/FfmpegService.java`

核心职责：
- 检测视频音频编码是否需要转码（EAC3/DTS/TrueHD/AC3 等）
- 启动 FFmpeg 进程，视频流直接 copy（`-c:v copy`），音频转 AAC（`-c:a aac -b:a 320k`）
- 输出到 `PipedOutputStream`，Controller 从 `PipedInputStream` 读取流式返回

关键命令：
```
ffmpeg -i input.mkv -c:v copy -c:a aac -b:a 320k -f mp4 -movflags +frag_keyframe+empty_moov pipe:1
```

### 3. 改造 streamVideo 端点

`VideoController.java` — `GET /{id}/stream`

流程：
1. 读取视频的 format/音频编码信息（用 ffprobe 或 ffmpeg 采集日志）
2. 如果音频编码是 EAC3/DTS/TrueHD/AC3 → 启动 FFmpeg 转码，返回 `text/plain` + chunked 流
3. 否则 → 直接返回原始文件（保持现有行为）

### 4. 新增 FfmpegProbeService（可选）

用 `ffprobe` 探测音频编码格式，决定是否需要转码。

或更简单的方案：根据文件扩展名 + 已知编码模式判断，不额外依赖 ffprobe。

## 需要修改/新增的文件

| 文件 | 操作 |
|------|------|
| `app/src/main/resources/application.yml` | 新增 `hub.ffmpeg.path`、`hub.video.transcode-audio` 配置 |
| `video/.../service/FfmpegService.java` | 新增，FFmpeg 进程管理和流式输出 |
| `video/.../controller/VideoController.java` | 改造 `streamVideo` 方法 |

## 验证
1. 播放 MKV + EAC3 视频 → 应有声音（AAC 转码）
2. 播放 MP4 + AAC 视频 → 直接透传，无转码
3. 拖动进度条 → 能正常 seek（FFmpeg 输出 frag_keyframe）
4. 停止播放 → FFmpeg 进程正常退出（不泄漏）
