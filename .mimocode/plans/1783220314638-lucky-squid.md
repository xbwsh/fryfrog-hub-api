# 恢复 FFmpeg 功能 + 内嵌打包方案

## 背景

用户要求恢复后端的音频转码和字幕转码功能，并参考 Jellyfin 的方式将 FFmpeg 打包进 Java 应用，用户无需手动安装。

## 方案概述

使用 `org.bytedeco:ffmpeg-platform` Maven 依赖，将跨平台 FFmpeg 二进制文件打包进 JAR。启动时自动提取到临时目录，通过 `ProcessBuilder` 调用。

## 任务步骤

### Step 1: 恢复 Maven 依赖

**文件**: `video/pom.xml`

添加回 bytedeco 依赖（仅 Linux x86_64，Docker 部署专用）：
```xml
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>ffmpeg</artifactId>
    <version>6.1.1-1.5.10</version>
</dependency>
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>ffmpeg</artifactId>
    <version>6.1.1-1.5.10</version>
    <classifier>linux-x86_64</classifier>
</dependency>
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>javacpp</artifactId>
    <version>1.5.10</version>
</dependency>
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>javacpp</artifactId>
    <version>1.5.10</version>
    <classifier>linux-x86_64</classifier>
</dependency>
```

> 仅引入 Linux x86_64 平台二进制，JAR 体积增加约 100MB。Docker 部署环境统一为 Linux x86_64。

### Step 2: 新建 FFmpegBinaryExtractor

**文件**: `video/src/main/java/com/fryfrog/hub/video/util/FFmpegBinaryExtractor.java`（新建）

功能：
- 从 bytedeco JAR 的 classpath 资源中提取 `ffmpeg` 和 `ffprobe` 二进制文件
- 提取到 `{java.io.tmpdir}/fryfrog-ffmpeg/{os-arch}/` 目录
- 已提取过则跳过（缓存机制）
- 返回 `record BinaryPaths(String ffmpeg, String ffprobe)`

平台映射：
| OS | Arch | 目录名 |
|----|------|--------|
| Windows | x86_64 | `windows-x86_64` |
| macOS | arm64 | `macosx-arm64` |
| macOS | x86_64 | `macosx-x86_64` |
| Linux | x86_64 | `linux-x86_64` |
| Linux | arm64 | `linux-arm64` |

## 接口文档

### GET /api/v1/video/{id}/media-info

获取视频技术信息。

**返回示例：**
```json
{
  "videoCodec": "h264",
  "videoCodecLong": "H.264 / AVC / MPEG-4 AVC / MPEG-4 part 10",
  "audioCodec": "aac",
  "audioCodecLong": "AAC (Advanced Audio Coding)",
  "audioChannels": 6,
  "audioSampleRate": "48000",
  "resolution": "1920x1080",
  "frameRate": 23.98,
  "bitrateKbps": 5000,
  "durationSeconds": 7380.0,
  "durationMinutes": 123,
  "format": "mov,mp4,m4a,3gp,3g2,mj2"
}
```

### GET /api/v1/video/{id}/subtitles

列出视频内嵌字幕轨道。

**返回示例：**
```json
[
  {
    "index": 2,
    "codec": "subrip",
    "language": "chi",
    "title": "简体中文",
    "default": true
  },
  {
    "index": 3,
    "codec": "ass",
    "language": "eng",
    "title": "English",
    "default": false
  }
]
```

### GET /api/v1/video/{id}/subtitle/vtt

将字幕转换为 WebVTT 格式。

**参数：**
- `index`（可选）— 内嵌字幕流索引
- `file`（可选）— 外挂字幕文件名

**返回：** `text/vtt; charset=utf-8` 格式的 WebVTT 内容

### POST /api/v1/video/{id}/audio/transcode

将不兼容的音频编码（eac3/dts/truehd/dca/mlp）转为 AAC。

**请求参数：**
- 无（自动检测不兼容编码）

**返回示例：**
```json
{
  "success": true,
  "data": {
    "originalCodec": "eac3",
    "transcodedPath": "/data/media/video/movie_transcoded.mp4",
    "streamUrl": "/api/v1/video/{id}/stream?transcoded=true"
  }
}
```

**行为：**
- 如果音频已兼容（aac/mp3/flac 等），直接返回 `"alreadyCompatible": true`
- 如果不兼容，使用 ffmpeg 转码：`ffmpeg -i {input} -c:v copy -c:a aac -b:a 192k {output}`
- 转码后文件保存在同目录下 `{baseName}_transcoded.{ext}`
- 返回流式播放路径供前端使用

## 验证方式

1. `mvn clean compile -DskipTests` — 编译通过
2. 启动应用，检查日志中是否输出 "FFmpeg 已就绪（内嵌版本）"
3. 调用 `GET /api/v1/video/{id}/media-info` 验证 ffprobe 分析
4. 调用 `GET /api/v1/video/{id}/subtitles` 验证字幕轨道列表
5. 调用 `GET /api/v1/video/{id}/subtitle/vtt?index=0` 验证字幕转换
6. 调用 `POST /api/v1/video/{id}/audio/transcode` 验证音频转码
7. 扫描新视频，验证技术参数（codec/resolution 等）被自动填充
8. Docker 构建验证：`docker build -t test .` 确认镜像包含内嵌 FFmpeg
