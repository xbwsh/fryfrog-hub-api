# 统一 Bangumi 刮削逻辑

## 目标

将漫画和电子书的 Bangumi 搜索 API 统一为 v0，消除 `BangumiService.searchManga()`（v1 旧版 API），并将两个模块中完全重复的工具方法提取到 `BangumiService` 中。

## 修改文件

### 1. `common/src/main/java/com/fryfrog/hub/common/service/BangumiService.java`

**搜索 API 统一：**
- 删除 `searchManga()` 方法（v1 旧版 API）
- 重命名 `searchV0Books()` → `search()`，作为唯一搜索入口
- `searchBooks(query, subType)` 保留，内部调用 `search()` 后按 subType 过滤
- `search()` 失败时不再需要回退到 `searchManga()`，直接返回空列表

**提取共享工具方法（从两个模块中重复的代码）：**

```java
// 卷号提取（漫画/电子书完全相同）
public static Integer extractVolumeFromRelatedName(String name)

// 构建卷号→关联条目映射（漫画/电子书完全相同）
public Map<Integer, RelatedSubject> buildVolumeSubjectMap(Integer subjectId)

// 搜索结果排序（漫画/电子书完全相同）
public static List<SearchResult> sortByPopularity(List<SearchResult> results)

// tags 转 genre 字符串（漫画/电子书完全相同）
public static String extractGenresFromTags(List<SearchResult.Tag> tags)

// 从 infobox 提取值（漫画/电子书完全相同）
public static String extractInfoboxValue(SubjectDetail detail, String key)
```

**新增封面下载方法（从两个模块中几乎相同的代码提取）：**
```java
public String downloadCover(SubjectDetail detail, String filePrefix, Path targetDir)
```
- 统一的封面 URL 选择逻辑（large → common）
- 统一的文件名模式 `{prefix}_cover.jpg`
- 统一的缓存检查和异常处理

### 2. `comic/src/main/java/com/fryfrog/hub/comic/service/MangaScrapeService.java`

- `searchFromBangumi()`: 改调 `bangumiService.search(query)` 替代 `bangumiService.searchManga(cleaned)`
  - 标题清洗逻辑保留（漫画特有的需求）
  - 清洗后无结果仍可用原始标题重试
- 删除 `extractVolumeFromRelatedName()` → 用 `BangumiService.extractVolumeFromRelatedName()`
- 删除 `buildVolumeSubjectMap()` → 用 `bangumiService.buildVolumeSubjectMap()`
- 删除 `extractInfoboxValue()` → 用 `BangumiService.extractInfoboxValue()`
- `downloadCoverFromBangumi()` / `downloadCover()`: 改用 `bangumiService.downloadCover()`
- `propagateSeriesMetadata()`: 使用共享的卷号映射和 infobox 提取方法

### 3. `ebook/src/main/java/com/fryfrog/hub/ebook/service/EbookBangumiScrapeService.java`

- `searchFromBangumi()`: 改调 `bangumiService.searchBooks(query, subType)` （已是如此，无需改动）
- 删除 `extractVolumeFromRelatedName()` → 用 `BangumiService.extractVolumeFromRelatedName()`
- 删除 `buildVolumeMap()` → 用 `bangumiService.buildVolumeSubjectMap()`
- `downloadCoverFromBangumi()`: 改用 `bangumiService.downloadCover()`
- tags→genre 转换：改用 `BangumiService.extractGenresFromTags()`

### 4. `ebook/src/main/java/com/fryfrog/hub/ebook/service/EbookMetadataScrapeService.java`

- `autoScrapeEbook()`: 改调 `bangumiService.searchBooks(query, "novel")` （已是如此，无需改动）

## 不变的部分

- 漫画的 `cleanTitleForSearch()` — 漫画特有的标题清洗，保留在 `MangaScrapeService`
- 漫画的 `saveBangumiCharacters()` — 漫画特有的角色抓取
- 漫画的 `moveComicToSeriesFolder()` — 漫画特有的文件移动
- 漫画的 `bindSeries` 开关 — 漫画特有的系列传播控制
- 电子书的 Open Library 回退 — 电子书特有的双源策略
- 两个模块各自的 `updateXxxFromBangumiDetail()` — 字段映射不同，保留各自实现
- REST API 路径和请求方式 — 不影响后端逻辑

## 实施顺序

1. 修改 `BangumiService`：删除 `searchManga()`，添加共享工具方法
2. 修改 `MangaScrapeService`：适配新的统一 API
3. 修改 `EbookBangumiScrapeService`：适配新的统一 API
4. 编译验证
