# 数据库重构：新建 MediaSeries 表，规范化漫画和电子书数据

## 目标

将漫画和电子书的系列信息独立建表，消除数据冗余，统一角色管理。

## 当前问题

- 系列信息（author/genre/seriesSummary 等）散落在每本单行本中
- 角色物理绑定到单本，逻辑上属于系列，用 JPQL 子查询 hack
- `ComicSeries`/`EbookSeries` DTO 在运行时用 `groupingBy` 动态聚合
- 每次刮削都可能重复下载系列封面、重复更新系列级字段

## 新数据模型

```
MediaSeries (新表)
├── id, title, author, genre, description, mediaType
├── metadataSource, metadataSourceId, rating, year
├── coverArtPath, serializationStart
├── favorite
├── @OneToMany → Comic[]
├── @OneToMany → Ebook[]
└── @OneToMany → MediaSeriesCharacter[]

Comic (修改)
├── 新增: series_id → MediaSeries (ManyToOne)
├── 旧 series 字符串列: 保留, insertable=false, updatable=false
└── 删除: characters 关系 (迁移到 MediaSeriesCharacter)

Ebook (修改)
├── 新增: series_id → MediaSeries (ManyToOne)
├── 旧 series 字符串列: 保留, insertable=false, updatable=false
└── 删除: characters 关系 (迁移到 MediaSeriesCharacter)

MediaSeriesCharacter (新表)
├── series_id → MediaSeries (ManyToOne)
├── name, originalName, role, description
├── source, sourceCharacterId
└── imageUrl, imagePath
```

## 实施步骤

### Step 1: 新建 MediaSeries 实体

**文件**: `common/src/main/java/com/fryfrog/hub/common/model/MediaSeries.java`

```java
@Entity
@Table(name = "media_series", indexes = {
    @Index(name = "idx_media_series_title", columnList = "title"),
    @Index(name = "idx_media_series_media_type", columnList = "mediaType")
})
public class MediaSeries extends BaseEntity {
    @Column(nullable = false)
    private String title;                    // 系列名
    private String author;                   // 作者
    private String genre;                    // 类型
    private String originalTitle;            // 原始标题
    @Column(length = 16)
    private String mediaType;                // "comic" | "ebook"
    private String metadataSource;           // bangumi / anilist
    private Integer metadataSourceId;        // 外部条目 ID
    private Double rating;
    private Integer year;
    @Column(columnDefinition = "TEXT")
    private String description;              // 系列简介
    private String coverArtPath;             // 系列封面本地路径
    private String serializationStart;
    private Boolean favorite = false;

    @OneToMany(mappedBy = "series", fetch = FetchType.LAZY)
    @OrderBy("volume ASC")
    private List<Comic> comics = new ArrayList<>();

    @OneToMany(mappedBy = "series", fetch = FetchType.LAZY)
    @OrderBy("volume ASC")
    private List<Ebook> ebooks = new ArrayList<>();

    @OneToMany(mappedBy = "series", cascade = CascadeType.REMOVE, fetch = FetchType.LAZY)
    private List<MediaSeriesCharacter> characters = new ArrayList<>();

    @JsonGetter("coverUrl")
    public String getCoverUrl() {
        return "/api/v1/media/series/" + getId() + "/cover";
    }

    public int getVolumeCount() {
        return (comics != null ? comics.size() : 0) + (ebooks != null ? ebooks.size() : 0);
    }
}
```

### Step 6: 数据迁移服务

**文件**: `common/src/main/java/com/fryfrog/hub/common/service/MediaSeriesMigrationService.java`

```java
@Service
@RequiredArgsConstructor
public class MediaSeriesMigrationService {
    private final MediaSeriesRepository seriesRepo;
    private final ComicRepository comicRepo;
    private final EbookRepository ebookRepo;
    private final ComicCharacterRepository comicCharRepo;
    private final EbookCharacterRepository ebookCharRepo;
    private final MediaSeriesCharacterRepository mediaCharRepo;

    @PostConstruct
    @Transactional
    public void migrate() {
        if (seriesRepo.count() > 0) return; // 已迁移过

        // 1. 收集所有 series 名
        Set<String> allSeries = new HashSet<>();
        comicRepo.findAllDistinctSeries().forEach(allSeries::add);
        ebookRepo.findAllDistinctSeries().forEach(allSeries::add);
        allSeries.remove(null);
        allSeries.remove("");

        // 2. 为每个 series 创建 MediaSeries
        for (String seriesName : allSeries) {
            MediaSeries ms = new MediaSeries();
            ms.setTitle(seriesName);
            ms.setMediaType(determineMediaType(seriesName));
            enrichFromExistingData(ms, seriesName);
            seriesRepo.save(ms);

            // 3. 更新外键
            comicRepo.updateSeriesRef(seriesName, ms.getId());
            ebookRepo.updateSeriesRef(seriesName, ms.getId());

            // 4. 迁移角色
            migrateCharacters(seriesName, ms);
        }
    }
}
```

### Step 7: 修改 MangaScrapeService

核心变更：
- `bindBangumi()`: 查找或创建 MediaSeries，设置 `comic.seriesRef = ms`
- `saveBangumiCharacters()`: 创建 MediaSeriesCharacter 而非 ComicCharacter
- `autoScrapeComic()`: 复用同系列 MediaSeries 的 metadataSourceId
- `rescrapeSeries()`: 参数改为 MediaSeries 对象
- 删除/废弃: `propagateSeriesMetadata()`（功能由 MediaSeries 统一管理）
- 删除/废弃: `moveComicToSeriesFolder()`（文件夹归类改用 MediaSeries.title）
