# 漫画分级刮削策略

## 目标

支持两种刮削模式：
1. **系列级刮削**：绑定一个卷时，自动将系列级元数据（作者、系列名、年份、类型、评分）同步到同系列所有卷，每卷保留自己的封面
2. **卷级刮削**：每卷独立绑定到各自的 Bangumi/AniList 条目（Bangumi 有每卷独立条目）

## 修改文件

### 1. `ComicRepository.java` — 新增查询方法

```java
List<Comic> findBySeriesIgnoreCase(String series);
```

用于查找同系列所有漫画。

### 2. `MangaScrapeService.java` — 核心逻辑

**新增方法 `propagateSeriesMetadata(Comic source)`**：
- 从 `source` 漫画提取系列级字段：`author`, `series`, `year`, `genre`, `rating`, `originalTitle`, `metadataSource`, `metadataSourceId`
- 用 `findBySeriesIgnoreCase(source.getSeries())` 查找同系列所有漫画
- 对每个非 source 的漫画：
  - 更新系列级字段
  - **不覆盖** `coverArtPath`（每卷保留自己的封面）
  - **不覆盖** `summary`（每卷保留自己的简介，如果是系列级刮削则用 source 的 summary 作为兜底）
  - 标记 `metadataSource` 和 `metadataSourceId`（表示已关联到该系列）
  - 移动文件到系列文件夹
  - 保存

**修改 `bindBangumi(comicId, bangumiId)` 签名**，新增参数 `boolean bindSeries`：
- 现有逻辑：绑定单个漫画（`bindSeries=false` 时直接返回）
- 新增逻辑：`bindSeries=true` 时，调用 `propagateSeriesMetadata(comic)` 同步到其他卷

**修改 `bindAnilist(comicId, anilistId)` 签名**，同上。

### 3. `ComicController.java` — 更新端点

修改两个 bind 端点，增加 `bindSeries` 参数：

```
POST /api/v1/comic/{id}/bangumi/bind?bangumiId=xxx&bindSeries=false
POST /api/v1/comic/{id}/anilist/bind?anilistId=xxx&bindSeries=false
```

`bindSeries` 默认 `false`，保持向后兼容。

## 流程图

### 系列级刮削 (bindSeries=true)

```
用户选择漫画A(卷1) + Bangumi Subject ID
  │
  ├─ 1. 绑定漫画A（现有逻辑）
  │     - 更新所有字段
  │     - 下载封面
  │     - 移动到系列文件夹
  │
  ├─ 2. propagateSeriesMetadata(漫画A)
  │     - 查找 findBySeries("系列名")
  │     - 对每个其他卷 B, C, D...
  │       ├─ 同步 author, series, year, genre, rating
  │       ├─ 保留 B 的 coverArtPath 和 summary
  │       ├─ 设置 metadataSource, metadataSourceId
  │       └─ 移动到系列文件夹
  │
  └─ 3. 返回漫画A
```

### 卷级刮削 (bindSeries=false)

```
用户选择漫画B(卷2) + Bangumi Subject ID (卷2的条目)
  │
  └─ 仅绑定漫画B，其他卷不受影响
```

## 验证方式

1. 扫描一个包含多卷的漫画目录
2. 用 `bindSeries=true` 绑定卷1到 Bangumi，验证：
   - 卷1 全部字段被更新
   - 卷2/3/... 的 author, series, year, genre, rating 被同步
   - 卷2/3/... 的 coverArtPath 保持不变（各自从文件提取的封面）
   - 所有卷都移动到同一文件夹
3. 用 `bindSeries=false` 绑定卷2到另一个 Bangumi ID，验证：
   - 仅卷2 被更新
   - 其他卷不受影响
4. 运行 `mvn test -pl comic` 确认现有测试通过
