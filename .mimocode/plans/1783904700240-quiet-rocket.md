# Plan: 全模块首页列表加分页 + 分组返回

## 目标

将四个模块（video/comic/ebook/music）的根列表端点从"平铺返回全部数据"改为"分组返回 + 分页"，首页不再一次性加载所有条目。

## 设计决策

### 分页方案

新建 `PageResponse<T>` DTO（common 模块），包装分页元数据：

```java
public class PageResponse<T> {
    private List<T> content;      // 当前页数据
    private int page;             // 当前页码（0-based）
    private int size;             // 每页大小
    private long totalElements;   // 总条目数
    private int totalPages;       // 总页数
}
```

Controller 层使用 Spring 的 `@RequestParam(defaultValue = "0") int page` 和 `@RequestParam(defaultValue = "20") int size` 做参数绑定（SQLite 不支持原生分页，用 `LIMIT/OFFSET` JPQL 实现）。

### 分组方案

根端点按"组"返回，每个组包含其下的所有子项：

| 模块 | 分组维度 | Group DTO | 子项 DTO | 现有类似结构 |
|------|----------|-----------|----------|-------------|
| Video | 系列 | `SeriesDTO`（已有） | `VideoDTO` | `GET /series` 已实现 |
| Comic | 系列 | `ComicSeries`（已有） | `ComicDTO` | `GET /series` 已实现 |
| Ebook | 系列 | `EbookSeries`（已有） | `EbookDTO` | `GET /series` 已实现 |
| Music | 专辑 | **新建 `AlbumGroup`** | `MusicTrack` | 无 |

### 根端点变更

| 模块 | 原端点 | 新返回类型 | 说明 |
|------|--------|-----------|------|
| Video | `GET /api/v1/video` → `List<VideoDTO>` | `PageResponse<SeriesDTO>` | 按系列分组 |
| Comic | `GET /api/v1/comic` → `List<ComicDTO>` | `PageResponse<ComicSeries>` | 按系列分组 |
| Ebook | `GET /api/v1/ebook` → `List<EbookDTO>` | `PageResponse<EbookSeries>` | 按系列分组 |
| Music | `GET /api/v1/music` → `List<MusicTrack>` | `PageResponse<AlbumGroup>` | 按专辑分组 |

### 搜索端点保持平铺

`/search/*` 端点返回平铺结果 + 分页（搜索结果不需要分组，用户要的是精确匹配）。

## 实施步骤

### Step 1: common 模块 — 新建 PageResponse

**文件**: `common/src/main/java/com/fryfrog/hub/common/dto/PageResponse.java`

```java
@Data
@NoArgsConstructor
@AllArgsConstructor
@Schema(description = "分页响应")
public class PageResponse<T> {
    private List<T> content;
    private int page;
    private int size;
    private long totalElements;
    private int totalPages;

    public static <T> PageResponse<T> of(List<T> content, int page, int size, long totalElements) {
        int totalPages = (int) Math.ceil((double) totalElements / size);
        return new PageResponse<>(content, page, size, totalElements, totalPages);
    }
}
```

### Step 2: music 模块 — 新建 AlbumGroup DTO

**文件**: `music/src/main/java/com/fryfrog/hub/music/dto/AlbumGroup.java`

```java
@Data
@NoArgsConstructor
@AllArgsConstructor
@Schema(description = "专辑分组")
public class AlbumGroup {
    private String artist;
    private String album;
    private Integer year;
    private List<MusicTrack> tracks;

    @JsonGetter("coverUrl")
    public String getCoverUrl() {
        // 返回第一首有封面的曲目的封面 URL
        if (tracks == null) return null;
        for (MusicTrack t : tracks) {
            if (t.getCoverArtPath() != null) return "/api/v1/music/" + t.getId() + "/cover";
        }
        return null;
    }
}
```
