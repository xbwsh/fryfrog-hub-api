# API 接口文档生成

**模块**: `$1`

## 任务

为 fryfrog-hub-api 的 `$1` 模块生成一份完整的 API 接口文档，用于前端 Flutter 团队对接。

## 要求

1. **扫描 controller 层**: 读取 `$1` 模块下所有 `@RestController` 类及其所有端点方法
2. **格式要求**: 每个端点包含以下内容：
   - HTTP 方法 + 完整路径（如 `GET /api/v1/music/{id}/stream`）
   - 功能说明（中文）
   - 请求参数（路径参数、查询参数、请求体 DTO 的字段说明）
   - 响应格式（JSON 结构示例 + 字段说明）
   - 可能的错误码/异常情况
3. **数据模型说明**: 本模块使用的主要 DTO/实体，标注字段类型和含义
4. **鉴权要求**: 标注哪些接口需要 Bearer Token
5. **特殊处理**: 如有流式响应、文件下载、Range 请求等特殊处理，详细说明

## 输出

生成一份 Markdown 文档，按功能分类组织端点（如：查询、详情、流播放、刮削、管理等），输出到 `docs/API_$1.md`。

## 风格

- 标题：中文
- 字段名和路径：英文原文
- 示例 JSON：包含实际数据
- 适用于 Flutter 前端直接参考
